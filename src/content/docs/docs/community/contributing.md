## Contributing to SCS

We welcome contributions to the Structured Context Specification.

:::tip[This is spec work]
SCS is v0.3 and actively evolving. If you're struggling with context management — or if
you disagree with our framing — that's exactly the contribution we need.
:::

## How to contribute

### Report issues

- [GitHub Issues](https://github.com/tim-mccrimmon/structured-context-spec/issues)
- Bug reports
- Feature requests
- Spec clarifications

### Join discussions

- [GitHub Discussions](https://github.com/tim-mccrimmon/structured-context-spec/discussions)
- Share use cases
- Propose improvements
- Ask questions

### Contribute code

- Plugin improvements (scs-vibe, scs-team)
- Validators and tooling
- Examples and templates
- Documentation

### Contribute bundles

- Standards bundles (HIPAA, SOC2, CHAI, etc.)
- Example domain bundles
- Industry-specific templates

## Governance

Content coming soon. See the [RFC process](/docs/community/rfc-process/) for how changes
to the spec are proposed and reviewed.

## Code of conduct

Be respectful and constructive. We're building this together.
