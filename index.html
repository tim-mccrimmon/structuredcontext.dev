Defining context for AI systems

Structured Context Specification (SCS) is an open, early-stage effort to define what context means for AI systems — distinct from prompts, retrieval, memory, and tools — and how that context can be represented as a structured, versioned artifact at runtime.

Models are increasingly static and interchangeable. Behavior emerges from the information they are allowed to see and assume at inference time. SCS exists to make that information explicit, inspectable, and debatable.

What this project is

SCS is a draft specification, a set of schemas, and a working space for people who care about reliability, debuggability, and boundaries in AI systems. It is intentionally incomplete and opinionated.

What this project is not

SCS is not a prompt framework, a RAG system, an agent runtime, or a governance product. Those may build on top of structured context — but they are not defined here.

The question we are exploring

If behavior emerges at runtime, what information should an AI system be allowed to assume?

Everything in SCS is an attempt to answer — or challenge — that question.
